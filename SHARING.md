# خطة مشاركة مصروفاتي مع الأصدقاء

> الهدف: مشاركة التطبيق مع أصدقاء لجمع feedback وأفكار، مع **عزل تام للبيانات** (كل صديق يشوف بياناته فقط) و**خصوصية كاملة** (أنا ما أشوف بياناتهم، وهم ما يلمسون الـ backend حقّي).

## القرار: الطريقة 1 — قالب جاهز ينسخه الصديق

رابط التطبيق واحد مشترك للجميع (الموقع الثابت على GitHub Pages). كل صديق يجهّز **شيت + backend خاص فيه** عبر نسخ قالب جاهز، ويربطه من إعدادات التطبيق.

### ليش هالطريقة؟
أي شيء يكتب في Google Sheet خاص لازم يملك **صلاحية تعديل** عليه (مربوط بالشيت أو الحساب اللي يشغّله عنده وصول). فما يكفي إن الصديق "يلصق رابط شيته" ويشتغل من فراغ — لأن الـ backend حقّي يشتغل بحسابي أنا وما عنده وصول لشيته. الطريقة 1 تحل هذا: كل صديق عنده backend خاص يشتغل بحسابه هو.

**البدائل المرفوضة:**
- backend مشترك يفتح الشيتات بالـ ID + الصديق يشارك شيته معي → يكسر الخصوصية (أقدر أشوف بيانات الجميع).
- كل صديق يسوي fork للتطبيق كامل → معقّد بدون فائدة (الموقع الثابت يكفي مشترك).

## خطوات الصديق (مرة وحدة)
1. يفتح **الشيت القالب** → File → **Make a copy** (تنتسخ معها السكربت).
2. Extensions → Apps Script → **Deploy → New deployment → Web app**:
   - Execute as: **Me**
   - Who has access: **Anyone**
3. ينسخ رابط الـ `/exec` ويلصقه في **إعدادات التطبيق** (حقل رابط الـ Web App).
4. خلص — يبدأ يسجّل مصاريفه. بياناته في شيته فقط.

## مهام التنفيذ (على التطبيق)
- [ ] تجهيز **شيت قالب** + نسخة `apps-script.gs` مدمجة فيه (المصدر موجود بالفعل بجذر المشروع).
- [ ] دليل خطوة-بخطوة (يفضّل مصوّر) لنشر الـ Web App — للصديق غير التقني.
- [ ] تحسين تسمية/رسائل حقل "رابط الـ Web App" في تبويب الإعدادات ليفهمها غير التقني.
- [ ] التطبيق يبدأ **فاضي بدون شيت افتراضي** للزائر الجديد — إزالة الربط التلقائي بشيتي من `js/config.js` / init في `js/app.js`.
- [ ] جعل **التمويل اختياري** — مخفي افتراضياً، يُفعَّل لمن يبغى يسجّل تمويله.

## 🔒 الحماية بالمفتاح السري — مُنفّذة في الكود (تحتاج خطوة نشر منك)

تم في الكود:
- [x] الـ backend (`apps-script.gs`): بوابة حماية ترفض أي طلب بدون مفتاح صحيح. المفتاح يُقرأ من **Script Property** اسمه `SECRET` (لا يُكتب في الكود أبداً).
- [x] الفرونت (`js/save.js`): دالة `appendKey` تُلحق المفتاح بكل طلب. المفتاح يُخزّن في `settings.webappKey` (متصفحك فقط).
- [x] حقل **"المفتاح السري"** في تبويب الإعدادات (`js/render.js`).
- [x] إزالة الرابط الافتراضي من `js/config.js` (`WEBAPP_DEFAULT = ''`).
- [x] رفع نسخة كاش الـ PWA (`sw.js` → v10).

### خطوات النشر المتبقّية (عليك أنت — في حسابك)
1. **ولّد مفتاح قوي** (مثلاً 32 محرف عشوائي — أي مولّد كلمات سر).
2. افتح محرّر الـ Apps Script المرتبط بالشيت → **Project Settings (⚙️) → Script Properties → Add** → الاسم `SECRET` والقيمة = المفتاح.
3. **انسخ كود `apps-script.gs` المحدّث** إلى المحرّر (فيه `getSecret` والبوابة).
4. **Deploy → Manage deployments → ✎ → New version → Deploy** (نسخة جديدة من نفس النشر).
5. في التطبيق → تبويب الإعدادات → الصق نفس المفتاح في حقل **"المفتاح السري"** → احفظ الروابط.

> ⚠️ ملاحظة: الرابط القديم موجود في **سجل git التاريخي**، فحذفه من `config.js` ما يمحيه. الحماية الفعلية هي المفتاح — بمجرد ضبط `SECRET` ونشر النسخة الجديدة، أي طلب بالرابط القديم بدون المفتاح يُرفض (`unauthorized`).
>
> 💡 المفتاح اختياري على مستوى الـ backend: لو `SECRET` فاضي، الحماية معطّلة (توافق مؤقت). فعّلها فوراً بضبط الخاصية.
